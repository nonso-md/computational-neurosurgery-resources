# Annotation Method

Every paper in this repository has been evaluated using a structured 
four-field annotation framework before being added to the repository

## The Four Columns

**Research Question**: What question the paper was trying to answer.

**Method**: The methodology used in the paper. Includes study design, 
dataset, algorithm, sample size, etc

**Finding**:  The conclusion of the paper.

**Gap**: Questions left unanswered, not addressed, 
or notable limitation.

## How this grows the repo

The full column annotation is maintained privately in Zotero. 
Only the gap field is surfaced publicly, compressed into a single 
line in the gap note column of each table row.

A paper is only added to the repository when the full annotation has been done.
Nothing gets here until it is read.

## The Bottom Gaps Section
The section is updated only when the same absence appears 
independently across multiple papers, mapping recurring patterns into research questions.
